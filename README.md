# 🚀 GitQuest: GitHub 저장소 검색 보드

GitHub의 방대한 오픈소스 생태계를 탐험하기 위한 실시간 검색 대시보드입니다. 키워드 하나로 가장 인기 있는 저장소의 핵심 정보를 즉시 확인하세요.

> **"AI가 코드를 만들어도, 최종 판단은 개발자가 한다."**  
> 본 프로젝트는 AI 코딩 어시스턴트와의 협업을 통해 제작되었으며, AI가 제안한 코드를 개발자가 직접 검토하고 리팩토링하여 완성도를 높였습니다.

---

## 1. 프로젝트 소개
GitHub REST API를 연동하여 사용자가 입력한 키워드에 맞는 저장소들을 검색하고, 별점(Stars) 순으로 정렬된 상위 6개의 결과를 카드 형태로 보여주는 웹 애플리케이션입니다.

## 2. 학습 목표
- **API 연동 실습**: 외부 API(`GitHub REST API`)를 호출하고 데이터를 처리하는 과정 학습
- **비동기 프로그래밍**: `async/await`와 `try/catch/finally`를 활용한 견고한 비동기 로직 구현
- **UI/UX 상태 관리**: 로딩, 검색 결과 없음, 에러 발생 등 다양한 상황에 맞는 UI 반응 설계
- **코드 리팩토링**: 중복을 제거하고 가독성을 높이는 함수화 과정 경험
- **버전 관리**: Git과 GitHub를 이용한 단계별 커밋 및 협업 워크플로우 숙달

## 3. 기술 스택
- **Frontend**: HTML5, CSS3, Vanilla JavaScript (ES6+)
- **API**: GitHub Search REST API
- **Version Control**: Git, GitHub
- **Styling**: 다크 모드 테마, CSS Grid (반응형 레이아웃)

## 4. 실행 방법
1. 저장소를 클론합니다: `git clone https://github.com/hong-vibe/20260511-Lecture-03.git`
2. `index.html` 파일을 브라우저로 엽니다.
3. 검색창에 검색어를 입력하고 엔터 또는 검색 버튼을 누릅니다.

## 5. 주요 기능
- **실시간 저장소 검색**: 별점순 내림차순 정렬 및 상위 6개 노출
- **스마트한 상태 처리**: 검색 중 버튼 비활성화, 로딩 메시지 표시, 결과 없음 안내
- **상세 정보 카드**: 저장소 이름, 설명, 주 사용 언어, 별점(Stars), 포크(Forks) 수 표시
- **반응형 디자인**: 데스크톱(3열), 태블릿(2열), 모바일(1열) 자동 레이아웃 최적화
- **사용자 편의성**: 검색어 앞뒤 공백 자동 제거(Trim), 엔터 키 검색 지원

## 7. AI 생성 결과 검토 기준
- **로직의 일관성**: `handleSearch` 함수가 입력-요청-출력의 흐름을 올바르게 제어하는가?
- **예외 처리**: 네트워크 에러나 데이터 부재 시 사용자에게 적절한 피드백을 주는가?
- **코드 가독성**: 초보자도 이해할 수 있는 명확한 변수명과 상세한 주석이 포함되었는가?
- **성능 및 보안**: `encodeURIComponent` 사용 및 불필요한 API 재호출 방지가 되었는가?

## 8. 수정 요청 내용
- **UI 개선**: 초기 Bootstrap 스타일에서 세련된 다크 모드로 전면 수정
- **코드 구조화**: `renderCards`를 `renderRepos`로 변경하고 `handleSearch`로 로직 통합
- **기능 강화**: Enter 키 검색 지원 및 검색 중 버튼 비활성화 추가
- **데이터 확장**: 기존 별점 외에 언어와 포크 수 정보 추가 노출

## 9. 배운 점
코드의 기능 구현만큼이나 **'상태 관리(State Management)'**의 중요성을 배웠습니다. 검색 중 버튼을 막거나 결과가 없을 때 안내를 주는 등의 작은 처리가 사용자 경험(UX)에 큰 차이를 만든다는 것을 체감했습니다. 또한 `async/await` 구조 내에서 `finally`를 사용해 로딩 상태를 해제하는 방식의 효율성을 익혔습니다.

## 10. 3줄 보고서
1. GitHub API를 활용하여 기능과 디자인이 완비된 검색 앱을 성공적으로 구축했습니다.
2. `try/catch/finally` 구조를 도입하여 예외 상황에서도 안정적인 서비스 운영이 가능합니다.
3. AI와의 협업을 통해 개발 생산성을 높이면서도, 직접 리팩토링을 수행하며 코드 품질을 확보했습니다.
