# 🚀 GitHub Scanner: 실시간 저장소 & 트렌드 검색 보드

GitHub의 방대한 오픈소스 생태계를 탐험하기 위한 실시간 검색 대시보드입니다. 키워드 하나로 가장 인기 있는 저장소의 핵심 정보를 즉시 확인하고, 하단의 트렌드 차트를 통해 개발 생태계의 인기 흐름을 파악해 보세요.

**🌐 Live Demo:** [https://hong-vibe.github.io/20260511-Lecture-03/](https://hong-vibe.github.io/20260511-Lecture-03/)
> **"AI가 코드를 만들어도, 최종 판단은 개발자가 한다."**  
> 본 프로젝트는 AI 코딩 어시스턴트와의 협업을 통해 제작되었으며, AI가 제안한 코드를 개발자가 직접 검토하고 리팩토링하여 완성도를 높였습니다.

---

## 1. 프로젝트 소개
GitHub REST API를 연동하여 사용자가 입력한 키워드에 맞는 저장소들을 검색하고, 별점(Stars) 순으로 정렬된 상위 6개의 결과를 카드 형태로 보여주는 웹 애플리케이션입니다. 특히 영문 저장소 설명을 한글로 즉시 번역해주는 기능을 탑재하여 편의성을 극대화했습니다.

## 2. 학습 목표
- **API 연동 실습**: 외부 API(`GitHub REST API`, `MyMemory API`)를 호출하고 데이터를 처리하는 과정 학습
- **비동기 프로그래밍**: `async/await`와 `try/catch/finally`를 활용한 견고한 비동기 로직 구현
- **UI/UX 상태 관리**: 로딩, 검색 결과 없음, 에러 발생 등 다양한 상황에 맞는 UI 반응 설계
- **성능 최적화**: 로컬 캐싱(`LocalStorage`)을 통한 중복 API 호출 방지 및 데이터 관리
- **브라우저 특성 이해**: `userAgent` 분석을 통한 조건부 기능 노출 및 브라우저별 특화 기능 구현

## 3. 기술 스택
- **Frontend**: HTML5, CSS3, Vanilla JavaScript (ES6+)
- **API**: GitHub Search REST API, MyMemory Translation API
- **Library**: Chart.js (트렌드 Bump Chart 렌더링)
- **Version Control**: Git, GitHub
- **Styling**: 다크 모드 테마, CSS Grid, 로고 타이포그래피 이펙트

## 4. 실행 방법
1. 저장소를 클론합니다: `git clone https://github.com/hong-vibe/20260511-Lecture-03.git`
2. `index.html` 파일을 브라우저로 엽니다.
3. 검색창에 검색어를 입력하고 엔터 또는 검색 버튼을 누릅니다.
4. (지원 브라우저) 검색 결과가 나오면 상단의 `🌐 번역` 버튼을 눌러 설명을 한글로 확인합니다.

## 5. 주요 기능
- **실시간 저장소 검색**: 별점순 내림차순 정렬 및 상위 6개 노출
- **개발 키워드 트렌드 차트**: 최근 4주간 가장 인기 있는 기술 스택의 순위 변동을 `Chart.js` Bump Chart로 시각화
- **차트 인터랙션**: 캔버스 네이티브 이벤트 핸들링을 통한 레이블 원클릭 자동 검색 연동
- **즉시 번역 (지원 브라우저)**: 카드 내 영문 설명을 확인 절차 없이 즉시 한글로 번역 (MyMemory API 활용)
- **번역 캐싱 시스템**: 한 번 번역된 내용은 `LocalStorage`에 저장하여 중복 API 호출 방지 및 속도 향상
- **스마트 브라우저 탐지**: 크롬/엣지 등 내장 번역을 지원하는 브라우저에서만 번역 버튼 노출
- **반응형 디자인 & 다크 모드**: 디바이스에 최적화된 레이아웃과 눈이 편안한 딥 블랙 다크 모드 및 로고 디자인

## 7. AI 생성 결과 검토 기준
- **로직의 일관성**: `handleSearch`와 `translateText` 함수가 데이터 흐름을 올바르게 제어하는가?
- **예외 처리**: 네트워크 에러나 API 한도 초과 시 사용자에게 적절한 피드백을 주는가?
- **코드 가독성**: 초보자도 이해할 수 있는 명확한 변수명과 상세한 주석이 포함되었는가?
- **성능 및 효율**: 캐싱 시스템을 통해 불필요한 네트워크 자원 낭비를 최소화했는가?

## 8. 수정 요청 내용
- **UI 및 기능 통합**: 다크 모드 적용 및 검색/번역 로직의 유기적 결합
- **번역 기능 고도화**: 브라우저 내장 번역 유도 방식에서 API를 통한 즉시 번역 방식으로 변경
- **API 최적화**: 이메일 파라미터(`de`)를 활용한 일일 무료 한도 확장(5만 자) 및 캐싱 도입
- **브라우저 필터링**: 번역 미지원 브라우저(비발디 등)에서 버튼을 숨기는 예외 처리 추가

## 9. 배운 점
코드 구현만큼이나 **'자원 관리'**와 **'사용자 경험'**의 조화가 중요하다는 것을 배웠습니다. 외부 API의 한도를 관리하기 위해 캐싱 로직을 직접 설계해보고, 브라우저별 특성을 파악하여 조건부 UI를 구성하는 과정을 통해 웹 표준과 실제 구현 사이의 차이를 깊이 있게 이해하게 되었습니다.

## 10. 3줄 보고서
1. GitHub API와 번역 API를 연동하여 실용적이고 세련된 다크 모드 검색 보드를 완성했습니다.
2. `Chart.js`를 활용하여 데이터 시각화(Bump Chart)와 검색 기능을 하나의 직관적인 대시보드 안에서 유기적으로 결합했습니다.
3. 네이티브 이벤트 핸들링 및 수학적 픽셀 거리 계산 로직을 구현하여 캔버스 차트 인터랙션의 사용자 경험(UX)을 완벽하게 통제했습니다.

## 11. 향후 개선 사항
- **검색 필터 추가**: 언어별 혹은 최소 별점 수 기준 필터 기능

- **다양한 번역 엔진**: MyMemory 외 다른 무료/유료 API 백업 연동
