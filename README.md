# 🚀 GitHub Scanner: 실시간 저장소 & 트렌드 검색 보드

GitHub의 방대한 오픈소스 생태계를 탐험하기 위한 실시간 검색 대시보드입니다. 키워드 하나로 가장 인기 있는 저장소의 핵심 정보를 즉시 확인하고, 하단의 트렌드 차트를 통해 개발 생태계의 인기 흐름을 파악해 보세요.

**🌐 Live Demo:** [https://hong-vibe.github.io/20260511-Lecture-03/](https://hong-vibe.github.io/20260511-Lecture-03/)

> **"AI가 코드를 만들어도, 최종 판단은 개발자가 한다."**  
> 본 프로젝트는 AI 코딩 어시스턴트와의 협업을 통해 제작되었으며, AI가 제안한 코드를 개발자가 직접 검토하고 리팩토링하여 완성도를 높였습니다.

---

## 1. 프로젝트 소개
GitHub REST API를 연동하여 사용자가 입력한 키워드에 맞는 저장소들을 검색하고, 별점(Stars) 순으로 정렬된 상위 6개의 결과를 카드 형태로 보여주는 웹 애플리케이션입니다. 더불어 하단에는 최근 4주간의 주요 개발 생태계 키워드 순위 변동 추이를 시각화한 트렌드 추적 차트를 제공하여, 개발 시장의 전반적인 흐름과 인기 기술을 한눈에 파악할 수 있도록 지원합니다. 필요 시 상단의 버튼 하나로 영문 저장소 설명을 한글로 번역해주는 기능도 탑재하여 사용 편의성을 극대화했습니다.

## 2. 주요 기능
- **실시간 저장소 검색**: 별점순 내림차순 정렬 및 상위 6개 핵심 정보 노출 (언어, 별점, 포크 수 등)
- **개발 키워드 트렌드 차트**: 최근 4주간 가장 인기 있는 기술 스택의 순위 변동을 `Chart.js` Bump Chart로 시각화
- **차트 원클릭 연동**: 캔버스 네이티브 이벤트 핸들링을 통해 차트 양옆의 텍스트 레이블 클릭 시 해당 키워드로 즉시 자동 검색 실행
- **API 기반 번역 기능**: 검색 결과 화면에서 `🌐 번역` 버튼을 클릭하면 `MyMemory API`를 통해 영문 설명을 한글로 번역
- **번역 캐싱 시스템**: 한 번 번역된 내용은 브라우저의 `LocalStorage`에 저장하여 중복 API 호출을 방지하고 빠른 렌더링 속도 보장
- **반응형 디자인 & 다크 모드**: 데스크톱(3열), 태블릿(2열), 모바일(1열)에 맞춰 자동으로 변경되는 레이아웃과 눈이 편안한 딥 블랙 다크 모드 테마 적용

## 3. 기술 스택
- **Frontend**: HTML5, CSS3, Vanilla JavaScript (ES6+)
- **API**: GitHub Search REST API, MyMemory Translation API
- **Library**: Chart.js (트렌드 시각화용)
- **Version Control**: Git, GitHub, GitHub Pages
- **Styling**: Vanilla CSS (CSS Grid, Flexbox, 다크 모드, 네온 텍스트 이펙트)

## 4. 실행 방법
1. 저장소를 클론합니다: `git clone https://github.com/hong-vibe/20260511-Lecture-03.git`
2. `index.html` 파일을 브라우저로 엽니다. (또는 Live Demo 링크를 클릭합니다)
3. 검색창에 검색어를 입력하고 엔터 또는 검색 버튼을 누릅니다. (또는 하단 차트의 키워드를 클릭합니다)
4. 검색 결과가 나오면 상단의 `🌐 번역` 버튼을 눌러 설명을 한글로 확인합니다.

## 5. 학습 목표 및 배운 점
- **API 연동과 데이터 시각화**: `GitHub API`로 텍스트 데이터를 가져오는 것을 넘어, `Chart.js`를 연동하여 복잡한 순위 데이터를 Bump Chart로 시각화하는 방법을 학습했습니다.
- **네이티브 이벤트 핸들링 통제**: `Chart.js`가 기본적으로 지원하지 않는 "여백 텍스트 클릭" 이벤트를 구현하기 위해, 캔버스 내 X/Y 좌표와 수학적 거리를 직접 계산하여 이벤트를 통제하는 고급 UX 설계 기법을 적용했습니다.
- **성능 최적화와 자원 관리**: 외부 API(MyMemory)의 일일 호출 한도를 고려하여 `LocalStorage` 기반의 자체 캐싱 로직을 구현함으로써 효율적인 네트워크 자원 관리의 중요성을 체감했습니다.
