# 🚀 GitQuest: GitHub 저장소 검색 보드

GitHub의 수많은 오픈소스 저장소를 간편하게 검색하고, 핵심 정보를 한눈에 파악할 수 있는 다크 모드 기반의 웹 대시보드입니다.

> **"AI가 코드를 만들어도, 최종 판단은 개발자가 한다."**  
> 본 프로젝트는 AI 코딩 어시스턴트와 개발자의 협업을 통해 만들어졌으며, 모든 로직과 디자인은 개발자의 검토를 거쳐 완성되었습니다.

---

## 1. 프로젝트 소개
GitHub REST API를 활용하여 실시간으로 저장소 데이터를 불러와 사용자에게 보여주는 웹 애플리케이션입니다. 검색어 입력만으로 가장 인기 있는(별점이 높은) 저장소 6개를 빠르게 확인할 수 있습니다.

## 2. 학습 목표
- **GitHub REST API**의 구조와 쿼리 파라미터 사용법 익히기
- **JavaScript 비동기 프로그래밍**(`async/await`, `fetch`) 이해
- **DOM 조작**을 통한 동적인 웹 페이지 구현
- **반응형 웹 디자인**(`CSS Grid`, `Media Queries`) 적용
- **Git & GitHub**를 활용한 프로젝트 관리 및 이력 기록

## 3. 기술 스택
- **언어**: HTML5, CSS3, Vanilla JavaScript (Pure JS)
- **API**: GitHub Search REST API
- **도구**: Git, GitHub, VS Code

## 4. 실행 방법
1. 본 저장소를 `git clone` 하거나 압축 파일을 다운로드합니다.
2. 폴더 내의 `index.html` 파일을 브라우저(Chrome, Edge 등)로 실행합니다.
3. 검색창에 원하는 키워드(예: react, python)를 입력하고 검색 버튼을 누르거나 엔터를 칩니다.

## 5. 주요 기능
- **실시간 검색**: GitHub API를 호출하여 실시간 데이터를 반영합니다.
- **다크 모드 UI**: 세련되고 눈이 편안한 다크 테마 디자인을 적용했습니다.
- **반응형 레이아웃**: 모바일(1열), 태블릿(2열), 데스크톱(3열) 환경에 최적화된 그리드를 제공합니다.
- **상태 알림**: 검색 중, 결과 없음, 에러 발생 등 현재 상태를 아이콘과 메시지로 알려줍니다.
- **상세 정보 표시**: 저장소 이름, 설명, 주 사용 언어, Star 수, Fork 수를 포함합니다.

## 7. AI 생성 결과 검토 기준
- **정확성**: GitHub API의 필드(`stargazers_count`, `forks_count` 등)가 올바르게 매핑되었는가?
- **가독성**: 초보자도 이해할 수 있도록 코드에 적절한 주석이 달려있는가?
- **효율성**: 중복 코드를 줄이기 위해 함수(`handleSearch`, `fetchRepos`)가 잘 분리되었는가?
- **사용자 경험(UX)**: 빈 검색어 입력이나 네트워크 에러 발생 시 적절한 안내가 나가는가?

## 8. 주요 수정 요청 내용
- 초기 단일 파일 구조를 HTML/CSS/JS로 **모듈화 분리** 요청
- 사용자 편의를 위한 **Enter 키 검색 지원** 및 중복 로직 통합
- 별점 외에 **언어 및 포크 수** 추가 표시 및 데이터 부재 시 예외 처리
- `alert` 팝업 대신 화면 내 **상태 섹션** 활용 유도

## 9. 배운 점
비동기 통신을 처리할 때 `.then()` 방식보다 `async/await`가 코드의 흐름을 파악하기에 훨씬 직관적이라는 것을 배웠습니다. 또한, UI 요소를 제어할 때 ID 명칭을 규칙적으로 관리하는 것이 자바스크립트 작성 시 실수를 줄여준다는 점을 체감했습니다.

3. AI와의 체계적인 협업 과정을 문서화하여 개발 이력을 투명하게 관리했습니다.
