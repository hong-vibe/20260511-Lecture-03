# GitHub 저장소 검색 기능 구현 프롬프트 기록

이 문서는 AI와 함께 GitHub REST API를 활용한 저장소 검색 기능을 구현하며 진행했던 대화와 요청 사항(프롬프트)들을 시간순으로 정리한 기록입니다.

## 1단계: API URL 설계 및 기본 Fetch 예시 요청
**요청 내용:**
GitHub REST API에서 저장소를 검색하는 요청 URL을 만들고 싶습니다.
조건:
1. 검색어는 사용자가 입력한 keyword를 사용합니다.
2. 저장소 검색 API를 사용합니다.
3. stars 기준으로 정렬합니다.
4. 내림차순으로 정렬합니다.
5. 결과는 6개만 가져옵니다.
6. fetch에서 사용할 수 있는 URL 문자열 예시를 보여주세요.

**진행 결과:**
조건에 맞는 GitHub Search API(`https://api.github.com/search/repositories`) URL의 구조와 간단한 `fetch` 사용 예제를 제공받았습니다.

URL을 JavaScript 함수 안에서 사용할 수 있도록 template literal 방식으로 다시 작성해 주세요.
조건:
1. 검색어 변수 이름은 keyword로 고정해 주세요.
2. 검색어는 encodeURIComponent로 처리해 주세요.

4. 초보자가 이해할 수 있도록 각 부분에 주석을 달아 주세요.

## 7단계: 시각적 상태 섹션(Status Section) 추가
**요청 내용:**
검색창 아래 섹션에 상태 섹션을 만들어주세요.

**진행 결과:**
단순 텍스트였던 상태 메시지를 시각적으로 돋보이는 `<section id="statusSection">` 박스로 만들고, 상태(검색 전, 진행 중, 완료, 결과 없음, 에러)에 맞춰 아이콘(💡, ⏳, ✅, 🤔, 🚨)과 배경색이 변하도록 업데이트했습니다.

---

## 8단계: 빈 검색어 예외 처리 개선
**요청 내용:**
검색어창에 빈칸인 상태로 엔터를 치면 상태 메시지 섹션에만 반응이 나와야하는데 팝업창도 뜹니다. 팝업창은 안뜨게 하세요

**진행 결과:**
빈 검색어 입력 시 동작하던 `alert()` 팝업을 제거하고, 대신 상태 섹션이 경고 모드(⚠️)로 변하면서 "검색어를 입력해주세요."라는 메시지를 띄우도록 `app.js`를 수정했습니다.

---

## 9단계: HTML/JS 구조 단순화 및 ID 명칭 통일
**요청 내용:**
결과 목록 영역의 id를 `repoList`로, 상태 메시지 영역의 id를 `statusMessage`로 고정하고, 입력창은 `searchInput`, 버튼은 `searchButton`으로 변경해 주세요. (JavaScript에서 선택하기 쉽게 구조 단순화)

**진행 결과:**
기존에 나뉘어 있던 상태 아이콘과 섹션을 `<div id="statusMessage">` 하나로 통합하여 HTML 구조를 단순화했습니다. 또한 버튼의 ID를 `searchButton`으로 변경하고, JS 코드도 하나의 DOM 요소로만 상태(메시지 및 이모지)를 제어하도록 전면 수정했습니다.

---

## 10단계: 다크 모드 기반 반응형 CSS 디자인 적용
**요청 내용:**
흰색 배경의 Bootstrap 느낌을 버리고 다크 모드 기반의 검색 보드로 CSS를 전면 재작성해 주세요. 검색창과 버튼은 상단에 배치하고, 그리드 레이아웃을 통해 모바일(1열), 태블릿(2열), 데스크톱(3열)에 반응하도록 만들어 주세요. 또한 초보자를 위해 상세한 주석을 달아주세요.

**진행 결과:**
`style.css`를 다크 테마(#121212 배경)로 전면 수정했습니다. `@media` 쿼리를 활용해 화면 너비에 따라 1~3열로 카드가 자동 정렬되도록 반응형 그리드를 구현했으며, 마우스 호버(Hover) 시 카드가 떠오르는 애니메이션과 네온 느낌의 포커스 효과를 더했습니다. 모든 CSS 속성 윗줄에 그 역할이 무엇인지 알 수 있게 친절한 주석을 추가했습니다.

---

## 11단계: 검색 버튼 클릭 이벤트 로직 보완 및 주석 강화
**요청 내용:**
순수 JavaScript로 검색 버튼 클릭 시 입력창의 값을 읽는 코드를 작성해 주세요. (조건: 앞뒤 공백 제거, 빈칸일 시 에러 메시지 표시, 값이 있으면 console.log 출력, 초보자용 주석 추가)

**진행 결과:**
`app.js`의 버튼 클릭 이벤트 리스너(`addEventListener`) 부분을 요구사항에 맞춰 수정했습니다. `trim()`을 사용한 공백 제거, 빈 문자열(`""`) 검사 로직을 초보자가 이해하기 쉽도록 번호를 매긴 상세한 한글 주석과 함께 작성했으며, 정상 입력 시 `console.log("검색어:", keyword)`로 값을 출력하도록 기능을 추가했습니다.
